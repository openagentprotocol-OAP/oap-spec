# Contributing to the Open Agent Protocol

Thank you for considering a contribution. OAP is an open standard governed through a public RFC process. This document explains how to engage productively.

## Ways to contribute

1. **Editorial.** Typos, formatting, and clarifications against the latest published draft. Submit a pull request directly.
2. **Examples and reference material.** Worked Manifests, Receipts, Decision Records under `examples/`. Submit a pull request.
3. **Schemas.** Improvements to JSON Schemas under `schemas/v1.0/`. Submit a pull request with rationale.
4. **Adapters.** New or improved adapters to other protocols. Place under `adapters/`.
5. **Substantive protocol changes.** Open an RFC. See `governance/RFC-PROCESS.md`.

## Pull request guidelines

1. Reference the section of the specification you are changing.
2. Keep editorial changes separate from substantive changes.
3. All text contributions are accepted under the Creative Commons Attribution 4.0 International License. All code contributions are accepted under the Apache License 2.0.
4. Sign your commits using the Developer Certificate of Origin (`Signed-off-by` line in commit message).
5. Use US English in new editorial text.

## Style

The specification is written in the formal register expected of an open standard. Avoid marketing language. Use MUST, SHOULD, MAY in all capitals only with the meaning given by RFC 2119 and RFC 8174. Do not use em dashes; prefer commas, parentheses, or restructured sentences.

## Communication

Substantive discussion happens on the public OAP mailing list and in tracked GitHub issues. Confidential reports of security vulnerabilities or ethics concerns are routed through the dedicated channels documented in `SECURITY.md` and the Foundation Whistleblower Program.

## Conduct

All participants are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md). Reports of misconduct go to the Foundation Conduct Committee.
